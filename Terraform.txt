us-east-1
	Copy security state file from local to bastion
	Databases: Launch MongoDB, MySQL, Redis, RabbitMQ. Apps cannot start without their DBs.
	ACM: Request and validate your SSL certificate (it takes a few minutes to issue).
	Backend ALB: Create the internal load balancer.
	Frontend ALB: Create the public load balancer (depends on ACM).
	SSM: Run terraform apply to sync to store all the new ARNs (ALB Listeners) and IDs.
	App Servers (Catalogue, User, etc.): Launch these now; they will "plug into" the Backend ALB and DBs.
	Frontend Servers: Launch these last. They "plug into" the Frontend ALB and point to the Backend ALB.
The Destruction Order (The Tear Down)
Destruction is the exact reverse. You must remove the "Consumers" before the "Producers."
	In Bation
	Frontend Servers: Delete these first to stop public traffic.
	App Servers: Delete the microservices. This "unplugs" them from the ALB and DBs.
	Frontend ALB: Delete the public entry point.
	Backend ALB: Delete the internal entry point.
	SSM: Run terraform apply (as we discussed) to clean up the "Ghost" ARNs.
	Databases: Delete the data tier.
	ACM: Delete the certificate.
	In local terraform destroy in bastion folder
	In local change nat_gateway_enable = true in infrastructure-live\dev\us-east-1\network\terraform.tfvars and run terraform 	apply
	Security: Delete the security groups last. (AWS won't let you delete an SG if a server is still using it).
