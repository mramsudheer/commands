DAST(Dynamic Application Security Testing)
=====================================
Tillnow we have seen static source code analysis, SAST(static application and security testing)SonarScan, Library Scan(Dependabot), Image Scan (Trivy).

DAST means testing our application while it is running to find security vulnerabilities before it goes live, or while it is in production. During a scan, the tool dynamically injects malicious payloads to attack the application URL. This process is completed before moving the project to production, or on a recurring basis every three months, using either internal tools or third-party services. Based on the findings, the team will verify and address the discovered vulnerabilities.

Veracode is the one of the DAST tool. It has free trail, but we need organization email.
